# Контекст GitHub и production-сервера

Проверено: 21 июля 2026 года.

Этот файл собирает инфраструктурный контекст из проекта `Взаимные подписки`, чтобы новый HH-бот использовал тот же GitHub-аккаунт и тот же сервер, но оставался полностью отдельным сервисом.

## GitHub

- Активный GitHub-аккаунт на этом компьютере: `romivchat`.
- Репозиторий бота взаимных подписок: `https://github.com/romivchat/vz_podpiski.git`.
- Исходный репозиторий HH-бота: `https://github.com/fikstt2/hh-ai-agent.git`.
- Репозиторий `https://github.com/romivchat/hh-ai-agent` создан 21 июля 2026 года как форк исходного проекта.
- Для HH-бота основной удалённый репозиторий (`origin`) должен быть `romivchat/hh-ai-agent`, а репозиторий автора (`fikstt2/hh-ai-agent`) нужно сохранить как `upstream` для получения будущих обновлений.
- Токены, `.env`, `state.json`, база данных и SSH-ключи нельзя добавлять в GitHub.

Рекомендуемая схема репозиториев:

```text
origin    -> https://github.com/romivchat/hh-ai-agent.git
upstream  -> https://github.com/fikstt2/hh-ai-agent.git
```

## Production-сервер

- SSH-алиас на этом компьютере: `vzbot`.
- Команда подключения: `ssh vzbot`.
- Запасная команда: `ssh -i ~/.ssh/server_key root@213.176.115.241`.
- Адрес сервера: `213.176.115.241`.
- Пользователь SSH: `root`.
- Общая папка проектов на сервере: `/opt/projects`.
- Существующий бот взаимных подписок: `/opt/projects/vz-podpiski`.
- Новый HH-бот должен находиться отдельно: `/opt/projects/hh-ai-agent`.

### Проверка доступа 21 июля 2026 года

- Алиас `vzbot` правильно указывает на `root@213.176.115.241` и файл `~/.ssh/server_key`.
- Сервер отклонил `~/.ssh/server_key` с ошибкой `Permission denied (publickey,password)`.
- Последние два запуска GitHub Actions для `romivchat/vz_podpiski` от 3 июля 2026 года завершились успешно, включая серверный деплой.
- На компьютере есть отдельный ключ `~/.ssh/vz_podpiski_github_actions`, но он предназначен для GitHub Actions и не используется для ручного входа без отдельного решения владельца.
- До восстановления ручного SSH-доступа нельзя выполнять установку HH-бота на production-сервер.

## Правила совместного размещения

- Не изменять и не останавливать `/opt/projects/vz-podpiski` при установке HH-бота.
- Использовать отдельные папку проекта, `.env`, базу, Telegram-токен и процесс запуска.
- Существующий бот работает в Docker. HH-бот в первой версии по плану работает через отдельный `systemd`-сервис и Python-окружение `.venv`.
- Имя нового сервиса: `hh-ai-agent.service`.
- Перед любым обновлением сохранять резервные копии `.env`, `state.json` и `agent.db`.
- Наличие GitHub Actions secrets в `vz_podpiski` не означает, что они доступны новому репозиторию: секреты GitHub задаются отдельно для каждого репозитория.
