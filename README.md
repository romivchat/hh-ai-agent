# HH.ru AI Agent с подтверждением откликов

ИИ-агент ищет вакансии на HH.ru, фильтрует их и готовит сопроводительные письма с помощью локальной нейросети. Бот работает на сервере, а Ollama и модель Qwen запускаются на Mac владельца. Каждая подходящая вакансия сначала приходит владельцу в Telegram. Без нажатия кнопки `Откликнуться` бот ничего не отправляет на HH.ru.

Эта версия основана на проекте [`fikstt2/hh-ai-agent`](https://github.com/fikstt2/hh-ai-agent) и добавляет обязательное ручное подтверждение каждого отклика.

## Фичи
- **Умный парсинг**: обходит Cloudflare и блокировки HH.ru за счет Playwright (сохраняет сессию авторизации, пробрасывает капчу в тг).
- **Двойная фильтрация**: хард-фильтр по стоп-словам + интеллектуальный анализ вакансии через локальную LLM (отсекает нерелевантные вакансии вроде "продажников" и "риелторов", даже если они всплыли в поиске по IT).
- **Сопроводительные без выдумок**: нейросеть выбирает только два направления вакансии, а письмо собирается из закрытого банка подтверждённых фактов.
- **Решение в Telegram**: для каждой вакансии доступны кнопки `Откликнуться`, `Изменить письмо` и `Пропустить навсегда`.
- **Очередь до 10 вакансий**: ожидающие решения хранятся бессрочно и сохраняются после перезапуска.
- **Защита от повторной отправки**: вакансия атомарно блокируется перед откликом, поэтому двойное нажатие не создаёт второй запрос.
- **Ответы работодателей**: новые сообщения из чатов HH.ru пересылаются в Telegram.
- **Работает 24/7**: бесконечный цикл с задержками (по умолчанию 30 минут) позволяет боту работать на вас круглосуточно.

## Требования
- Python 3.10+
- Установленная на Mac [Ollama](https://ollama.com/) с моделью `qwen3:4b-instruct`.
- Защищённый обратный SSH-туннель с Mac на сервер. Порт Ollama не открывается в интернет.

## Настройка (Гайд для себя и потомков)

### 1. Установка зависимостей
Склонируйте репозиторий и установите библиотеки:
```bash
pip install -r requirements.txt
playwright install chromium
```

### 2. Настройка окружения
Скопируйте `.env.example` в `.env` и добавьте свои значения:
```ini
TG_BOT_TOKEN="ВАШ_ТОКЕН_ОТ_BOTFATHER"
TG_USER_ID="ВАШ_TELEGRAM_ID"
MAX_PENDING_JOBS="10"
HH_SUBMISSION_ENABLED="false"
OLLAMA_URL="http://localhost:11434/api/generate"
OLLAMA_MODEL="qwen3:4b-instruct"
APPLICANT_NAME="ВАШЕ_ИМЯ"
GITHUB_URL="https://github.com/romivchat"
CANDIDATE_PROFILE_PATH="/opt/projects/hh-ai-agent/candidate_profile.json"
TARGET_RESUME_NAMES="PRODUCT_MANAGER|PRODUCT_OWNER|CPO"
MY_RESUME_SUMMARY="ОПИСАНИЕ_ОПЫТА_НАВЫКОВ_И_ПОЖЕЛАНИЙ"
```
*(Свой ID можно узнать через бота @userinfobot в ТГ).*

### 3. Настройка поиска под себя
Откройте `config.py` и подстройте логику под своё резюме:
- `SEARCH_QUERIES` — список продуктовых ролей для поиска.
- `APPLICANT_NAME` — имя для подписи в конце письма.
- `GITHUB_URL` — ссылка на ваш GitHub.
- `TARGET_RESUME_NAMES` — названия резюме на HH через `|`, в порядке приоритета.
- `MY_RESUME_SUMMARY` — описание опыта для фильтрации вакансий.
- `CANDIDATE_PROFILE_PATH` — закрытый JSON с позиционированием, senior-контекстом и только подтверждёнными фактами для писем.

Создайте закрытый профиль из примера:

```bash
cp candidate_profile.example.json candidate_profile.json
chmod 600 candidate_profile.json
```

`candidate_profile.json` не попадает в GitHub. Поле `forbidden_terms` блокирует названия текущего работодателя и его продукта. Qwen не пишет письмо свободно: она выбирает два `focus_ids`, после чего код собирает текст длиной 150–250 слов из одобренных формулировок.

Если обязательные значения не заполнены, бот завершится с понятной ошибкой и не начнёт обрабатывать вакансии.

### 4. Первый запуск (Авторизация)
При самом первом запуске скрипт откроет видимое окно браузера:
```bash
python main.py
```
Вам нужно **вручную войти в свой аккаунт HH.ru** (ввести код из SMS/почты). Как только вы войдете, скрипт сохранит куки в файл `state.json`. После этого можете закрыть скрипт (Ctrl+C).

### 5. Запуск
Запускайте скрипт заново:
```bash
python main.py
```
Теперь браузер запустится в скрытом режиме и начнёт собирать вакансии. Напишите Telegram-боту `/start`. Подходящие вакансии будут приходить с письмом и кнопками решения.

Команда `/pending` повторно показывает все вакансии, которые ожидают решения. Они не исчезают по времени.

Во время первоначальной настройки оставьте `HH_SUBMISSION_ENABLED="false"`. В этом режиме можно проверять очередь, кнопки и изменение писем, но кнопка `Откликнуться` физически не отправит данные на HH.ru. Значение `true` разрешается установить только после полной проверки.

## Как сервер использует Ollama на Mac

Mac сам устанавливает зашифрованное SSH-соединение с сервером. На сервере появляется только локальный адрес `127.0.0.1:11434`, который ведёт к Ollama на Mac. В интернет этот порт не публикуется, поэтому в настройках бота остаётся обычный адрес `http://localhost:11434/api/generate`.

Mac должен быть включён, не находиться в режиме сна и иметь доступ в интернет. Обычный VPN можно оставить включённым. Если Mac или туннель временно недоступны, бот приостанавливает анализ до следующего цикла. Текущая вакансия не получает статус `filtered`, а пустое шаблонное письмо не создаётся.

## Как принимается решение

- `Откликнуться` — бот открывает конкретную вакансию, подставляет сохранённое письмо и отправляет отклик.
- `Изменить письмо` — бот ждёт новый текст одним сообщением, сохраняет его и снова показывает кнопки. Само изменение письмо не отправляет.
- `Пропустить навсегда` — вакансия сохраняется со статусом `skipped` и больше не появляется в предложениях.

Когда ожидают решения 10 вакансий, поиск временно не добавляет новые. После любого решения место освобождается автоматически.

## Гарантия ручного подтверждения

Поисковый цикл не содержит команд нажатия кнопок отклика или отправки. Реальное нажатие на HH.ru находится в отдельном методе и вызывается только обработчиком кнопки `Откликнуться` из Telegram. Перед отправкой база дополнительно проверяет, что вакансия всё ещё ожидает решения.

## Как изменить локацию поиска?
По умолчанию бот ищет вакансии сначала в Москве (`&area=1`), а затем удаленку по всей РФ (`&area=113&schedule=remote`).
Вы можете изменить `area` (код региона) в файле `hh_client.py` внутри метода `search_and_queue`.
