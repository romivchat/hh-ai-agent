# HH.ru AI Agent с подтверждением откликов

ИИ-агент ищет вакансии на HH.ru, фильтрует их и готовит сопроводительные письма с помощью локальной нейросети. Каждая подходящая вакансия сначала приходит владельцу в Telegram. Без нажатия кнопки `Откликнуться` бот ничего не отправляет на HH.ru.

Эта версия основана на проекте [`fikstt2/hh-ai-agent`](https://github.com/fikstt2/hh-ai-agent) и добавляет обязательное ручное подтверждение каждого отклика.

## Фичи
- **Умный парсинг**: обходит Cloudflare и блокировки HH.ru за счет Playwright (сохраняет сессию авторизации, пробрасывает капчу в тг).
- **Двойная фильтрация**: хард-фильтр по стоп-словам + интеллектуальный анализ вакансии через локальную LLM (отсекает нерелевантные вакансии вроде "продажников" и "риелторов", даже если они всплыли в поиске по IT).
- **Кастомные сопроводительные письма**: нейросеть читает ваше резюме и текст вакансии, затем готовит персонализированный черновик.
- **Решение в Telegram**: для каждой вакансии доступны кнопки `Откликнуться`, `Изменить письмо` и `Пропустить навсегда`.
- **Очередь до 10 вакансий**: ожидающие решения хранятся бессрочно и сохраняются после перезапуска.
- **Защита от повторной отправки**: вакансия атомарно блокируется перед откликом, поэтому двойное нажатие не создаёт второй запрос.
- **Ответы работодателей**: новые сообщения из чатов HH.ru пересылаются в Telegram.
- **Работает 24/7**: бесконечный цикл с задержками (по умолчанию 30 минут) позволяет боту работать на вас круглосуточно.

## Требования
- Python 3.10+
- Установленный [Ollama](https://ollama.com/) с загруженной моделью (по умолчанию `llama3` или `llama3:8b`).

## Настройка (Гайд для себя и потомков)

### 1. Установка зависимостей
Склонируйте репозиторий и установите библиотеки:
```bash
pip install -r requirements.txt
playwright install chromium
```

### 2. Настройка окружения
Скопируйте `.env.example` в `.env` и добавьте свои значения:
```ini
TG_BOT_TOKEN="ВАШ_ТОКЕН_ОТ_BOTFATHER"
TG_USER_ID="ВАШ_TELEGRAM_ID"
MAX_PENDING_JOBS="10"
OLLAMA_URL="http://localhost:11434/api/generate"
OLLAMA_MODEL="llama3"
```
*(Свой ID можно узнать через бота @userinfobot в ТГ).*

### 3. Настройка поиска под себя
Откройте `config.py` и подстройте логику под своё резюме:
- `SEARCH_QUERIES` — список запросов (например, "Python backend").
- `TARGET_RESUME_NAME` — точное название вашего резюме на HH (чтобы бот выбирал правильное из списка).
- `MY_RESUME_SUMMARY` — подробное текстовое саммари ваших навыков. **Это критически важно**, так как на основе этого текста ИИ будет писать сопроводительные письма. Опишите свой стек, проекты и пожелания максимально подробно.

### 4. Первый запуск (Авторизация)
При самом первом запуске скрипт откроет видимое окно браузера:
```bash
python main.py
```
Вам нужно **вручную войти в свой аккаунт HH.ru** (ввести код из SMS/почты). Как только вы войдете, скрипт сохранит куки в файл `state.json`. После этого можете закрыть скрипт (Ctrl+C).

### 5. Запуск
Запускайте скрипт заново:
```bash
python main.py
```
Теперь браузер запустится в скрытом режиме и начнёт собирать вакансии. Напишите Telegram-боту `/start`. Подходящие вакансии будут приходить с письмом и кнопками решения.

Команда `/pending` повторно показывает все вакансии, которые ожидают решения. Они не исчезают по времени.

## Как принимается решение

- `Откликнуться` — бот открывает конкретную вакансию, подставляет сохранённое письмо и отправляет отклик.
- `Изменить письмо` — бот ждёт новый текст одним сообщением, сохраняет его и снова показывает кнопки. Само изменение письмо не отправляет.
- `Пропустить навсегда` — вакансия сохраняется со статусом `skipped` и больше не появляется в предложениях.

Когда ожидают решения 10 вакансий, поиск временно не добавляет новые. После любого решения место освобождается автоматически.

## Гарантия ручного подтверждения

Поисковый цикл не содержит команд нажатия кнопок отклика или отправки. Реальное нажатие на HH.ru находится в отдельном методе и вызывается только обработчиком кнопки `Откликнуться` из Telegram. Перед отправкой база дополнительно проверяет, что вакансия всё ещё ожидает решения.

## Как изменить локацию поиска?
По умолчанию бот ищет вакансии сначала в Санкт-Петербурге (`&area=2`), а затем удаленку по всей РФ (`&area=113&schedule=remote`).
Вы можете изменить `area` (код региона) в файле `hh_client.py` внутри метода `search_and_queue`.
